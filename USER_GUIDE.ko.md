# Lexisub 사용 설명서

> 영상 → 한국어 자막 자동 생성 + 도메인 용어집 적용 + 줄별 검토/편집까지.
> Apple Silicon Mac에서 로컬로 동작. 인터넷은 첫 실행 시 모델 다운로드에만 필요.

---

## 1. 빠르게 시작하기 (3분)

### 1-1. 다운로드 + 설치 (DMG 권장 — 가장 쉬움)

1. [Releases 페이지](https://github.com/existmaster/lexisub/releases/latest)에서 **`Lexisub-vX.Y.Z-macos-arm64.dmg`** 다운로드
2. DMG 파일 더블클릭 → Finder 윈도우가 열립니다
3. **Lexisub.app을 옆에 있는 Applications 폴더로 드래그**
4. Finder의 **응용 프로그램** 폴더로 이동
5. 처음 한 번만: **Lexisub 아이콘을 우클릭**(Control+클릭) → **열기**

   "확인되지 않은 개발자" 경고가 뜨면 [열기]를 한 번 더 누르면 됩니다. 다음부터는 일반적으로 더블클릭만으로 실행됩니다.

> ffmpeg는 앱 안에 자동 포함되어 있어 별도 설치 불필요. 터미널 명령도 필요 없습니다.

### 1-2. 첫 실행 — 모델 다운로드

처음 영상을 처리하면 약 **4GB의 ML 모델**이 자동 다운로드됩니다 (`~/.cache/huggingface/`에 저장).

| 모델 | 크기 | 역할 |
|---|---|---|
| Whisper large-v3-turbo | ~1.5GB | 음성 → 텍스트 |
| Gemma 3 4B 4-bit | ~2.5GB | 영어 → 한국어 번역, 용어 추출 |

다운로드는 한 번만. 이후엔 즉시 동작합니다.

---

## 2. 메인 화면 구조

세 개 탭:

| 탭 | 용도 |
|---|---|
| **영상 처리** | 영상 파일 → 한국어 자막 .srt + 자막 포함 .mkv 생성 |
| **PDF 라이브러리** | 도메인 PDF에서 전문 용어를 자동 추출 |
| **용어집** | 추출된 용어 검토/편집/승인 (승인된 용어만 영상 번역에 적용) |

---

## 3. 권장 워크플로우

### A. 가장 단순한 사용 (도메인 용어집 없이)

1. **영상 처리** 탭 → 영상 파일 드래그&드롭 (또는 [파일 찾기])
2. [자막 생성 시작] 클릭
3. 진행: 음성 추출 → 음성 인식 → 번역 → 자막 mux → 완료
4. 영상 옆에 두 파일 생성:
   - `<영상이름>.ko.srt` — 한국어 자막
   - `<영상이름>.subbed.mkv` — 한국어 자막 트랙이 포함된 영상 (재인코딩 없음, VLC에서 자막 토글 가능)

처리 시간: 영상 길이의 약 **0.3~0.5배** (5분 영상 → 약 2분).

### B. 도메인 용어집 활용 (의학·재활·격투기 등 전문 영상에 권장)

1. **PDF 라이브러리** 탭 → [PDF 추가] → 도메인 교재 PDF 선택 (다중 가능)
2. 원본 언어는 **자동 감지** 기본값 그대로
3. 백그라운드에서 LLM이 본문에서 전문 용어를 추출 (페이지당 약 30초)
4. **용어집** 탭으로 이동 → pending 상태로 추가된 용어 검토
5. 잘못된 번역은 [편집]으로 수정, 어색한 건 [선택 삭제]
6. 만족스러우면 [보류 전부 승인] (또는 개별 더블클릭으로 토글)
7. 영상 처리 탭에서 영상 처리 → **승인된 용어가 자막에 강제 적용**

## 4. 용어집 탭 — 핵심 기능

### 4-1. 컬럼

| 컬럼 | 의미 |
|---|---|
| 언어 | 원어 언어 (en, ko, pt 등) |
| 원어 용어 | 원본 텍스트의 표기 (예: "Heel Strike") |
| 한국어 번역 | 자막에 적용될 한국어 (예: "발꿈치 접지") |
| 분류 | 기술/해부학/의학/개념/장비/인명/기타 |
| 상태 | **approved** (적용됨) / **pending** (검토 필요) / **rejected** |
| 근거 | **본문** (PDF에 한·영 병기됨) / **추론 ⚠** (LLM이 추측) / **사용자** (직접 편집됨) / **CSV** |
| 출처 | 어느 PDF의 어느 페이지에서 추출됐는지 |

> 🔑 **핵심**: "근거" 컬럼이 "추론 ⚠"인 행은 LLM이 자체 지식으로 채운 것이라 검증 권장. 헤더에 "⚠ 추론 N개"가 표시됩니다.

### 4-2. 행 동작

| 동작 | 결과 |
|---|---|
| 행 선택 (단일) | 우측 디테일 패널에 정의·문맥·출처 표시 |
| 행 더블클릭 | 승인 ↔ 보류 토글 |
| 행 선택 + [편집] | 모달 다이얼로그에서 한국어 번역, 분류, 정의, 상태 수정 |
| 행 선택 + Delete (또는 [선택 삭제]) | 다중 선택 후 일괄 삭제 |
| [선택 승인] | 다중 선택 일괄 승인 |
| [보류 전부 승인] | 모든 pending → approved (검토 부담 줄임) |
| [출처 없는 보류 정리] | PDF가 모두 제거되어 출처 사라진 보류 용어 일괄 청소 |

### 4-3. 편집 다이얼로그

수정 가능한 필드:
- **한국어 번역** (가장 자주 수정할 부분)
- **분류** (드롭다운 + 자유 입력)
- **상태**
- **정의** (1-2 문장 한국어 정의. 비워두면 정의 없음)

읽기 전용:
- 원어, 언어 (UNIQUE 키 보호 — 변경하려면 삭제 후 다시 추가)

저장 시 자동으로 **근거 = 사용자**로 승격됩니다.

## 5. 자막 편집 — fallback과 오역 빠르게 보정

영상 처리 후 결과 카드에서 [**자막 편집**] 버튼을 누르면 자막 편집 창이 뜹니다.

### 5-1. 자동 하이라이트

| 색상 | 의미 |
|---|---|
| 🔴 빨강 배경 | 번역 누락 — 영어 원문이 그대로 남음 (LLM이 첫 청크에서 형식을 못 잡으면 발생, 보통 영상 시작 1-2분) |
| 🟡 노랑 배경 | 음역 의심 — 한글과 알파벳이 붙음 (예: "Tyl러", "MMA러닝") |
| **굵게** | 본인이 편집한 줄 |

### 5-2. 효율적 보정 흐름

1. 상단 [✓ **의심 줄만 보기**] 체크 → OK 줄 숨김, 빨강·노랑만 표시
2. 영어 원문 컬럼 보면서 한국어 셀 더블클릭 → 직접 입력
3. 줄당 약 5-10초면 충분
4. 끝나면 [**저장 + .mkv 재mux**] → 1-3초 후 .mkv 갱신

> 💡 5분 영상에서 fallback이 ~10줄 + 음역 의심 ~4줄 정도. **약 2-3분이면 보정 완료**.

### 5-3. 저장 모드

| 버튼 | 동작 |
|---|---|
| **.srt만 저장** | `<영상>.ko.srt` 만 갱신 (외부 플레이어용 자막 파일만 필요할 때) |
| **저장 + .mkv 재mux** | `.ko.srt` + `<영상>.subbed.mkv` 둘 다 갱신. ffmpeg 스트림 복사라 빠름 |

## 6. PDF 라이브러리 탭

### 6-1. 작업 흐름

1. [PDF 추가] → 다중 선택 가능
2. 원본 언어는 **자동 감지** (langdetect로 PDF 첫 페이지의 주 언어 판단)
3. 진행률 표시. 페이지당 약 30초
4. 끝나면 "용어집 탭에서 검토하세요" 안내

### 6-2. 스캔 PDF 처리

PDF가 이미지 스캔이라 텍스트 레이어가 없으면 lexisub가 본문을 읽지 못합니다. OCR이 필요합니다.

```bash
# OCR 도구 설치 (한 번만)
uv sync --extra ocr

# OCR 실행 (페이지당 약 0.4초, 354페이지 기준 약 2.5분)
uv run python scripts/ocr_pdf.py "<scanned.pdf>"
```

OCR 결과는 `<scanned>.ocr.pdf` — 이걸 PDF 라이브러리에 추가합니다.

> Apple Vision OCR (macOS 네이티브) 사용. 한국어/영어 모두 정확도 높음. 다만 도표·일러스트 위주의 PDF는 본문 텍스트가 적어 추출량이 제한적입니다.

### 6-3. PDF 제거

1. PDF 라이브러리 탭에서 행 선택 (다중 가능)
2. [선택 제거] 클릭
3. 다이얼로그:
   - **PDF 제거 + 보류 용어 정리**: 그 PDF에서만 추출된 보류(pending) 용어도 청소 (CSV로 가져온 승인 용어는 보존)
   - **PDF만 제거**: PDF만 삭제, 용어는 그대로 남음 (다른 PDF에서도 추출됐을 가능성)

## 7. CSV 용어집 가져오기 / 내보내기

이미 가지고 있는 용어집이나 다른 사람이 만든 용어집을 가져올 수 있습니다.

CSV 형식:
```csv
source_lang,source_term,ko_term,category
en,guard pass,가드 패스,기술
en,armbar,암바,기술
pt,passagem de guarda,가드 패스,기술
```

용어집 탭의 [CSV 가져오기]로 임포트. 가져온 항목은 자동으로 **승인 + 근거=CSV** 상태가 됩니다.

`demos/glossaries/` 폴더에 BJJ/스트라이킹/일반 그래플링 예시 CSV가 있습니다 (소스에서 빌드한 경우만).

## 8. 자주 묻는 질문

### Q1. 왜 영상 처음 1-2분 자막이 영어 그대로 나오나요?

작은 LLM(Gemma 3 4B 4-bit)이 첫 청크에서 출력 형식을 안정적으로 따르지 못하는 한계입니다. 3회 재시도 후 **원문 그대로 두는 것이 깨진 텍스트보다 안전**하다는 판단으로 그렇게 동작합니다.

해결책: [자막 편집] 화면에서 빨강 배경 줄을 직접 입력 (영상 시작 부분이라 보통 5-10줄, 2-3분이면 충분).

### Q2. 인명이 "Tyl러" 같이 깨져요

영어 인명을 한국어로 음역할 때 LLM이 자주 실패합니다. 두 가지 해결:
1. 자막 편집에서 노랑 배경 줄을 직접 수정 ("타일러"로)
2. 용어집에 등록 (`source_term: Tyler`, `ko_term: 타일러`, 분류: 인명) → 다음 영상부터 자동 적용

### Q3. 의학 용어 번역이 부정확한 경우가 있어요

작은 LLM이 의학 도메인의 반대 개념 쌍 (eccentric ↔ concentric, eversion ↔ pronation 등)을 가끔 뒤집습니다. 용어집 탭의 **근거 컬럼**에서 "추론 ⚠"인 항목을 우선 검토하세요. 본문 근거(=PDF에 한·영 병기)인 항목은 보통 정확합니다.

### Q4. 모델 다운로드가 너무 느려요

기본 다운로드 출처는 huggingface.co입니다. 한국에서 느릴 수 있습니다. 미러 사용:

```bash
HF_ENDPOINT=https://hf-mirror.com open Lexisub.app
```

또는 영구 설정:
```bash
echo 'export HF_ENDPOINT=https://hf-mirror.com' >> ~/.zshrc
```

### Q5. 처리 중 메모리가 부족해요 (OOM)

8GB Mac에서 다른 무거운 앱(Chrome 탭 다수, Docker 등)과 동시에 돌리면 빡빡할 수 있습니다. 영상 처리 시작 전 다른 앱을 종료하세요. STT와 LLM은 순차 로드라 동시 메모리 사용은 ~3-4GB이지만 OS 캐시 등 여유가 필요합니다.

### Q6. 영상이 1시간 이상이에요

1시간 영상은 약 23분 처리 (8GB Air 기준). 2시간이면 약 46분. 시간이 길수록 첫 청크 fallback 비율이 상대적으로 작아져 자막 편집 부담이 줄어듭니다.

### Q7. 자막 색상/폰트 바꿀 수 있나요?

현재는 SRT 표준 형식이라 플레이어가 렌더링합니다 (VLC, 인피니티 등에서 자막 옵션으로 폰트/크기/색상 조정 가능). 향후 ASS 자막으로 스타일 임베드는 미정.

### Q8. 영상이 한국어인데 어떻게 되나요?

Whisper가 한국어로 감지하면 그대로 한국어 자막을 만듭니다 (번역 단계 사실상 무동작). 한국어 영상의 자막 추출용으로도 사용 가능합니다.
